Design Process and its justification

 

The design process for this task started with thinking about how exactly to visualize the map. I created the map using the path function and fed the json file to create lines and make the map by joining all the points. Next was to add the Pumps to the map which was done by creating circles using the coordinates given in the pumps csv file. I made the radius of the pumps = 10 so that pumps can stand out from the data points for the deaths. The color of the pumps was chosen as grey to make it different from the death points. Next, death locations were added to the map. To differentiate between male and female, the color of the circle was chosen as the parameter to visualize. Male deaths were marked by blue whereas Female deaths were marked by pink color. To segregate the deaths for different age groups, the radius of the circle was chosen as the parameter to indicate the age group. A bigger circle means older age groups. Tool tip is also added to the map to tell about the gender and age group of the person who died. Legend is present below the map to tell the viewer about the gender and age of the person. Moreover, the zoom functionality is also added to the map, to zoom and pan the map. However, the zoom that is added zooms the whole group inside which the map is present. The zoom window is not limited to a fixed window. The zooming and panning go indefinitely wherever the viewer wants to take the map. This was an issue I was unable to resolve.

 

 

To indicate the deaths for each day, I first thought of making a bar plot. But the bar map that I plotted had many bars and was not a good visualization to indicate in a timeline. I thought of replacing this with a line chart with circles to indicate deaths for each day. If you hover over the circle, there is a tooltip that indicates the date and the number of deaths. Also, hovering over the points in the timeline will randomly select the number of deaths on the map and make it five times bigger. This is an element of interactivity between both maps. The number of points that highlight on the map is equal to the number of deaths on the selected day in the timeline. However, the number of deaths does not show up on the map if the number of deaths is less than 20. I tried to figure out why this is not working. I tried printing the array which was dynamically generated when you hovered over the timeline, but those points were not highlighted in the map when the death count was less than 20.

 

 

 

To indicate deaths for different age and gender groups, I thought of making two bar plots - one for age groups and one for gender. But that I felt was very redundant. Then, I got the idea of making a side-by-side bar plot. In this plot, I made 6 age groups and had 2 side-by-side bars for both genders in each of the groups. This looked way better than 2 bar plots so I chose this one. Hovering over each of the bars will show a tooltip indicating deaths and gender.
